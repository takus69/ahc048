### 20250530

- AHC048開始。頑張るぞ！
- 問題理解
  - 絵具をK種類(4-20)持っている
  - N×N(20×20) のパレットを使って、H(1,000)の色を順に作成して、1グラムずつ提出する
  - 以下の4操作できる
    - 操作1: 1グラムずつ絵具を出す
    - 操作2: ウェルから作成した色を1グラム提出する
    - 操作3: ウェルの絵具を1グラム破棄する
    - 操作4: 仕切りを操作する(仕切りがなくなると色は混ざる)
  - 操作2は、ちょうどH回実施する
  - 操作1は、コストD(10-10,000)かかる
  - 操作3,4は、コストがかからないが、1-4すべて操作の合計は、T(4,000-64,000)回までしか行えない
  - 操作1をV回行ったとし、提出した絵具の誤差(三次元のユークリッド距離)の和をEとすると、スコアは、1-D(V-H)+round(10,000E)
    - 操作2の回数が少ない方が良いし、誤差が少ない方が良い
    - コストDが多いときはVが支配的、Dが少ないときはEが支配的になるか？
    - 誤差の平均を0.5とすると、誤差項の値は 10,000×0.5×1,000 = 5,000,000
    - V=10,000とすると、操作回数の項は、D×10,000 程度、つまり、100,000-100,000,000
      - 4種類の操作が同じ回数ずつ発生するとして、DT/4 が5,000,000程度を超えると、ターン数を抑えた方が良い
      - なんなら、1種類の近い絵具を提出するだけが一番いいかも
  - 目標色の生成における各絵の具のグラム数を可視化
    - 1グラム未満: 632回
    - 2グラム未満: 232回
    - 3グラム未満: 86回
    - 4グラム未満: 31回
    - 5グラム未満: 12回
    - 6グラム未満: 4回
    - 7グラム未満: 2回
  - 1グラム未満が多いので、1グラムずつ混ぜたような色が多い？
  - 文脈がありそうなので、ビームサーチ系がよさそう
  - 1色作るときの誤差が最小となる作り方はどうなるか？
    - 以下の2つを最小化したい
      - 3次元ベクトルの線形結合したベクトルと目的の色のベクトルの差を最小化
      - 上記線形結合の係数は整数で、係数の合計を最小化
  - 1色作るときの最適化、残った色を含めて次の色を作る最適化がありそう
